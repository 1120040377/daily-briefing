## 💻 前端资讯

**1. 《State of JavaScript 2025》调研报告深度解读上线 InfoQ**（发布日期：3月，2026）

InfoQ 发布了对 2025 年 JavaScript 生态调研结果的深度分析。核心发现：TypeScript 已彻底"赢得战役"，40% 开发者完全切换到 TypeScript；Vite 满意度高达 98%，而 Webpack 仅剩 26%；React 使用率仍最高（83.6%），但满意度有所下滑；Rust 构建工具 Rolldown 使用率从 1% 跃升至 10%。截至 2025 年底，近 30% 代码已由 AI 辅助生成。

🔗 [InfoQ 原文](https://www.infoq.com/news/2026/03/state-of-js-survey-2025/) · [2025.stateofjs.com](https://2025.stateofjs.com/en-US)

> 💡 **AI 点评：** Rolldown 的爆发式增长暗示未来 JavaScript 构建层将出现"Rust 化"趋势，Vite 本身也在朝这个方向演进。对团队决策者的建议：现在仍在用 Webpack 的项目迁移 Vite 的 ROI 极高；AI 辅助编码已成日常，需同步建立代码审查规范防止质量滑坡。

**2. CSS 锚点定位（Anchor Positioning）正式跨浏览器 Baseline 可用**（发布日期：2026年初）

随着 Firefox 147 的支持上线，CSS Anchor Positioning 已成为 Baseline Newly Available 特性，Chrome 125+、Edge 125+、Safari 26.1+ 均已支持。该特性允许开发者将弹窗、下拉菜单、Tooltip 等 UI 元素直接锚定到目标元素上定位，无需再依赖 JavaScript 计算坐标。

🔗 [MDN Web Docs](https://developer.mozilla.org/en-US/docs/Web/CSS/Guides/Anchor_positioning) · [Can I use](https://caniuse.com/css-anchor-positioning)

> 💡 **AI 点评：** Anchor Positioning 是近年来最实用的 CSS 新特性之一，可以直接替代大量依赖 Popper.js、Floating UI 的组件定位逻辑，减少 JavaScript 包体积。跨浏览器支持完成后，组件库作者应优先考虑迁移，普通项目新增浮层组件时也可直接采用原生方案。

**3. JavaScript 生态：Astro 向 Next.js 发起元框架挑战**（发布日期：2026年持续趋势）

《State of JavaScript 2025》及多份技术分析报告均指出，Astro 正在"主攻"Next.js 的市场地位。Astro 凭借出色的性能表现（静态优先、Island 架构）和更低的认知复杂度吸引大批开发者，而 Next.js 尽管使用率仍达 59%，但在满意度上持续承压，17% 用户持负面态度，且生成的评论数量最多——多为批评声音。

🔗 [InfoQ 调研](https://www.infoq.com/news/2026/03/state-of-js-survey-2025/) · [LogRocket 趋势分析](https://blog.logrocket.com/8-trends-web-dev-2026/)

> 💡 **AI 点评：** 元框架战争进入下半场，选型逻辑从"能用吗"转向"用得爽吗"。对新项目，内容驱动型网站优先考虑 Astro；需要大量动态交互和服务端渲染的全栈应用，Next.js 生态成熟度仍有优势但需接受其复杂性。
