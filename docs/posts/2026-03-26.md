## AI 资讯

**1. Anthropic 为 Claude Cowork 推出「电脑控制」功能，可直接操作 Mac 桌面**（发布日期：3月24日）

Anthropic 宣布三大功能更新：Claude Code 新增 auto mode（AI 安全分类器自动审批常规操作）、Claude Cowork 获得电脑控制能力（可直接操控 macOS 键盘和鼠标）、以及 Claude Code Channels（通过 Discord/Telegram 控制编程 Agent）。用户可从手机端通过 Dispatch 功能远程分配任务给桌面端 Claude。目前仅限 Claude Pro 和 Max 订阅用户使用。
🔗 [CNBC](https://www.cnbc.com/2026/03/24/anthropic-claude-ai-agent-use-computer-finish-tasks.html) | [WinBuzzer](https://winbuzzer.com/2026/03/25/anthropic-claude-code-cowork-auto-mode-computer-use-xcxwbn/)

> 💡 **AI 点评：** 这标志着 AI Agent 从"对话式"正式迈入"操作式"时代。Cowork 的电脑控制能力意味着非技术用户也能享受 Agent 自动化红利，配合 Dispatch 的手机-桌面联动，AI 助手正在成为真正的"数字员工"。开发者应关注 MCP 生态和 Agent 工具链的演进。

**2. Anthropic 起诉五角大楼案持续发酵，法官质疑军方"黑名单"决定**（发布日期：3月25日）

联邦法官表示五角大楼将 Anthropic 列入供应链风险名单的决定"看起来是试图摧毁该公司"。此前 Anthropic 拒绝让五角大楼无限制使用其 AI 技术，坚持安全限制。超过 30 名来自 OpenAI 和 Google DeepMind 的员工（包括 Google 首席科学家 Jeff Dean）提交法庭意见书支持 Anthropic。
🔗 [Al Jazeera](https://www.aljazeera.com/economy/2026/3/25/anthropics-case-against-the-pentagon-could-open-space-for-ai-regulation) | [CNBC](https://www.cnbc.com/2026/03/24/anthropic-lawsuit-pentagon-supply-chain-risk-claude.html)

> 💡 **AI 点评：** 这场诉讼可能成为 AI 行业监管的分水岭事件。Anthropic 坚持安全底线的立场获得行业广泛支持，说明"负责任 AI"不仅是口号。此案结果将深刻影响 AI 公司与政府合作的边界和规则。

**3. OpenAI 今日移除旧版 Deep Research 模式，推进 GPT-5.4 生态建设**（发布日期：3月26日）

OpenAI 于今日（3月26日）移除 ChatGPT 旧版 Deep Research 模式。此前 GPT-5.4 已于 3月5日发布，包括 Standard、Thinking 和 Pro 三个版本，支持高达 105 万 token 上下文。GPT-5.4 mini 和 nano 于 3月19日发布。同时，Codex for Students 向美国和加拿大大学生开放，每人可领取 $100 额度。
🔗 [Releasebot](https://releasebot.io/updates/openai/chatgpt) | [OpenAI News](https://openai.com/news/)

> 💡 **AI 点评：** 旧版 Deep Research 的移除意味着 OpenAI 正在全面切换到 GPT-5.4 架构。Codex for Students 是争夺下一代开发者的战略举措。对开发者来说，建议尽快迁移到 GPT-5.4 API，利用其更长的上下文窗口和更强的推理能力。

## 前端资讯

**1. TypeScript 6.0 正式发布，严格模式默认开启**（发布日期：3月23日）

Microsoft 正式发布 TypeScript 6.0，这是基于当前 JavaScript 代码库的最后一个大版本。主要变化包括：strict 模式默认启用、默认 module 切换为 esnext、默认 target 变为 ES2025。同时，基于 Go 语言重写的 TypeScript 7.0 编译器已"极其接近完成"，预计将带来 10 倍编译速度提升。
🔗 [Microsoft DevBlogs](https://devblogs.microsoft.com/typescript/announcing-typescript-6-0/) | [InfoWorld](https://www.infoworld.com/article/4149659/typescript-6-0-arrives.html)

> 💡 **AI 点评：** strict 默认开启对新项目是好消息，但存量项目迁移需要注意兼容性。TS 7.0 用 Go 重写编译器是里程碑式的决定——10 倍速度提升将彻底改变大型 TypeScript 项目的开发体验。建议团队尽早测试 TS 6.0 升级。

**2. WebStorm 2026.1 正式发布：原生 TypeScript 引擎、AI Agent 集成**（发布日期：3月25日）

JetBrains 发布 WebStorm 2026.1，亮点包括：基于服务端的 TypeScript 引擎、适配 TypeScript 6/7 的配置变更、React use memo/use no memo 指令高亮、Angular 21.x 模板新语法支持、Vue TypeScript 插件升级到 3.1.8、以及 AI 聊天中集成 Junie、Claude Agent 和 Codex 等多种 AI Agent。
🔗 [JetBrains Blog](https://blog.jetbrains.com/webstorm/2026/03/webstorm-2026-1/)

> 💡 **AI 点评：** WebStorm 全面拥抱 AI Agent 生态是 IDE 发展方向的缩影——未来的 IDE 不只是代码编辑器，更是 AI 协作平台。原生 TypeScript 引擎适配 TS 6/7 也说明工具链正在快速跟进语言变化。

**3. Svelte 三月月报：新增 createContext、TrustedHTML 支持**（发布日期：3月25日）

Svelte 官方发布三月月报，主要更新包括：Svelte 新增 createContext 支持程序化组件实例化、{@html} 表达式支持 TrustedHTML、HTML 标签内可写注释、服务端错误边界。SvelteKit Netlify 适配器有破坏性变更。Svelte CLI (sv@0.12.0) 将 better-auth 纳为官方插件。State of JS 2025 调查中 Svelte 继续保持响应式框架好评度第一。
