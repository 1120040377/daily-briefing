## AI 资讯

**1. Anthropic 意外泄露下一代模型「Claude Mythos」，性能实现"阶梯式飞跃"**（发布日期：3月26-27日）

Anthropic 因安全疏忽，在公开可访问的数据存储中意外曝光了约 3,000 份内部资产，其中包括代号为「Mythos」（另一版本为「Capybara」）的下一代 AI 模型信息。Anthropic 发言人确认该模型代表了"阶梯式的性能飞跃"，在软件编程、学术推理和网络安全等测试中取得了远超 Claude Opus 4.6 的成绩。泄露文件同时警告该模型可能大幅提升网络安全风险。

🔗 [Fortune](https://fortune.com/2026/03/26/anthropic-says-testing-mythos-powerful-new-ai-model-after-data-leak-reveals-its-existence-step-change-in-capabilities/) | [The Decoder](https://the-decoder.com/anthropic-leak-reveals-new-model-claude-mythos-with-dramatically-higher-scores-on-tests-than-any-previous-model/)

> 💡 **AI 点评：** 这次泄露本身就是一面镜子——即使是最重视安全的 AI 公司，也可能犯低级错误。Mythos 的性能如果属实，意味着 AI 能力的天花板再次被抬高，行业竞争将更加白热化。开发者应关注其正式发布后的 API 定价和安全策略。

**2. Google Gemini 推出「Import Memory」工具，支持从 ChatGPT/Claude 迁移记忆和对话**（发布日期：3月26日）

Google 在 Gemini Drop（2026年3月）中推出了「Import Memory to Gemini」功能，用户可以将 ChatGPT、Claude、Copilot 等 AI 服务的个性化记忆和聊天记录迁移至 Gemini。同批更新还包括音乐生成模型 Lyria 3 Pro、Gemini 3.1 Flash Live 开发者预览版，以及 Google TV 上的 Gemini 新功能。

🔗 [Google Blog](https://blog.google/innovation-and-ai/products/gemini-app/switch-to-gemini-app/) | [TechCrunch](https://techcrunch.com/2026/03/26/you-can-now-transfer-your-chats-and-personal-information-from-other-chatbots-directly-into-gemini/) | [9to5Google](https://9to5google.com/2026/03/26/gemini-import/)

> 💡 **AI 点评：** Google 这招"搬家工具"直接瞄准了用户迁移成本——AI 产品的护城河正在从"模型能力"转向"用户数据粘性"。对开发者来说，这预示着 AI 应用需要从一开始就考虑数据可移植性标准。

**3. Mistral 发布开源语音合成模型 Voxtral TTS，3秒即可克隆声音**（发布日期：3月26日）

法国 AI 公司 Mistral 发布了 4B 参数的文本转语音模型 Voxtral TTS，支持英语、法语、德语等 9 种语言，仅需 3 秒音频即可实现声音克隆，首次音频延迟仅 90ms。模型以 CC BY NC 4.0 许可在 Hugging Face 开放权重，API 定价 $0.016/千字符，直接对标 ElevenLabs 和 OpenAI。

🔗 [Mistral AI 官方](https://mistral.ai/news/voxtral-tts) | [TechCrunch](https://techcrunch.com/2026/03/26/mistral-releases-a-new-open-source-model-for-speech-generation/) | [VentureBeat](https://venturebeat.com/orchestration/mistral-ai-just-released-a-text-to-speech-model-it-says-beats-elevenlabs-and)

> 💡 **AI 点评：** Mistral 的开源策略继续搅动市场——4B 参数+开放权重意味着边缘设备部署成为可能。对于构建语音助手或客服系统的开发者来说，这是一个极具性价比的选择，但 NC 许可限制了商业用途，需注意合规。

**4. OpenAI 宣布关闭 Sora 视频应用，迪士尼 10 亿美元合作泡汤**（发布日期：3月24日）

OpenAI 宣布关闭上线仅 6 个月的 AI 视频生成应用 Sora。该应用峰值时每日推理成本约 1500 万美元，但总计应用内购收入仅约 210 万美元。下载量从 11 月的 333 万骤降至 2 月的 113 万。与迪士尼关于角色授权的合作也随之搁浅。CEO Sam Altman 提到了即将推出的新模型「Spud」。

🔗 [TechCrunch](https://techcrunch.com/2026/03/24/openais-sora-was-the-creepiest-app-on-your-phone-now-its-shutting-down/) | [CNN](https://edition.cnn.com/2026/03/24/tech/openai-sora-video-app-shutting-down) | [CNBC](https://www.cnbc.com/2026/03/24/openai-shutters-short-form-video-app-sora-as-company-reels-in-costs.html)

> 💡 **AI 点评：** Sora 的关闭证明了"技术领先≠产品成功"。每日 1500 万美元的推理成本暴露了 AI 视频生成的经济不可行性。这对整个 AI 消费级产品敲响了警钟——如果单位经济模型跑不通，再酷的技术也会被叫停。

**5. Anthropic 为 Claude Code 推出 AutoDream 记忆整理功能**（发布日期：3月24-27日）

Anthropic 为 Claude Code 推出了实验性功能 AutoDream，模拟生物睡眠中的记忆巩固机制，在空闲时自动执行四阶段循环：剪枝过时条目、合并重复信息、刷新结构化格式、压缩记忆索引至 200 行以内。该功能通过服务端 feature flag 控制，正在逐步灰度发布。

🔗 [TechCrunch](https://techcrunch.com/2026/03/24/anthropic-hands-claude-code-more-control-but-keeps-it-on-a-leash/) | [Geeky Gadgets](https://www.geeky-gadgets.com/claude-autodream-memory-files/)

> 💡 **AI 点评：** AutoDream 解决了 AI 编程助手的核心痛点——长期使用后记忆文件膨胀导致性能下降。"让 AI 像人一样做梦整理记忆"这个设计理念很有想象力。对重度 Claude Code 用户来说，这意味着跨 session 的上下文质量将显著提升。

## 前端资讯

**1. Chrome 147 Beta 发布**（发布日期：3月11日*——接近时间窗口边缘*）

Chrome 147 Beta 版本发布。此前 Chrome 146 已于 3月10日正式推出，带来了滚动触发动画（scroll-triggered animations）、Scoped Custom Element Registry（防止自定义元素命名冲突）和 Sanitizer API（帮助构建防 XSS 应用）。

🔗 [Chrome for Developers](https://developer.chrome.com/blog/chrome-147-beta)

> 💡 **AI 点评：** Sanitizer API 是前端安全的重大利好——原生浏览器级别的 XSS 防护意味着开发者不再需要依赖 DOMPurify 等第三方库。Scoped Custom Element Registry 则解决了微前端架构中的老大难问题。

**3. CSS 原生 Masonry 布局和 @mixin 提案持续推进**（发布日期：3月）

CSS 原生瀑布流布局（masonry）通过 `display: masonry` 或 `grid-template-rows: masonry` 方式持续推进标准化。同时 CSS 原生 `@mixin` 提案允许定义可复用的声明块（支持参数和条件逻辑），以及 `reading-flow` 属性控制 flex/grid 子元素的键盘导航和屏幕阅读器顺序。

🔗 [LogRocket Blog](https://blog.logrocket.com/css-in-2026/)

> 💡 **AI 点评：** CSS @mixin 原生化是继嵌套语法之后又一重磅特性——这意味着 Sass/Less 预处理器的核心卖点正在被浏览器逐一吞噬。前端工程师可以开始评估在新项目中减少对 CSS 预处理器的依赖。

🔗 [Chrome for Developers](https://developer.chrome.com/blog/new-in-devtools-146)
